# 项目开发需求文档

> 本文档用于记录、维护和跟踪 `cnAgentOS` 项目的开发需求。
>
> 维护原则：
> 1. 由人类与 AI 共同维护。
> 2. 业务需求与非业务需求统一记录。
> 3. 需求应尽量可拆分、可验收、可追踪。
> 4. 后续开发应结合 `readme.md` 中的项目架构说明进行实现。
> 5. 前端组件必须本地化使用，不引用互联网 CDN 资源。
> 6. 需求文档是长期维护文档，新增功能、优化项、缺陷修复、非功能要求都应记录。

---

## 1. 项目目标

本项目目标是逐步构建一个面向业务场景的 Web 系统，最终演进为：

- **AI 智能瞭望系统**
- **智能问数系统**
- **管理后台 + 用户侧门户**

当前阶段重点是：

- 保持现有 Tornado MVC 架构不变
- 继续完善登录体系与后台基础框架
- 为后续 AI 能力、问数能力、数据采集能力、管理能力预留结构
- 使用本地化前端组件统一页面风格和交互能力

---

## 2. 当前技术栈

当前技术栈如下：

- Python 3.12
- Tornado 6.5.5
- SQLite3
- HTML5
- CSS
- JavaScript

已纳入并要求本地化使用的前端组件：

- Bootstrap 5.3.8 Dist
- Font Awesome Free 5.15.4 Web
- layui 2.13.6

对应要求：

- 组件压缩包已放置在 `app/static/dist`
- 解压后必须以本地静态资源方式使用
- 页面开发时不得引用互联网 CDN 资源
- 后续页面应优先使用本地静态资源统一加载
- 组件路径和引用方式应便于后续维护和替换

---

## 3. 已实现能力

当前项目已实现并可复用的能力：

### 3.1 登录功能

- 用户名密码登录
- 表单参数校验
- 密码校验
- 登录成功写入 secure cookie
- 登录失败提示错误信息
- 登录后跳转首页

### 3.2 登录态控制

- 通过 `BaseHandler.get_current_user()` 读取登录态
- 未登录访问受保护页面会跳转登录页
- 首页已受认证保护

### 3.3 退出功能

- 清除登录 cookie
- 返回登录页

### 3.4 SQLite 用户存储

- 用户信息持久化存储到 SQLite
- 密码采用哈希 + 盐值方式保存

### 3.5 基础页面骨架

- 已存在基础模板 `base.html`
- 已存在登录页和首页模板
- 已存在基础 CSS 与 JS 资源文件

## 4. 需求跟踪规则

### 4.1 需求编号规范

建议使用如下编号方式：

- `REQ-F-001`：功能需求
- `REQ-NF-001`：非功能需求
- `REQ-TECH-001`：技术/架构需求
- `REQ-UI-001`：页面/UI 需求
- `REQ-DATA-001`：数据需求
- `REQ-BUG-001`：缺陷修复需求

### 4.2 需求状态

每一条需求都应具备状态：

- `待确认`
- `待开发`
- `开发中`
- `待验证`
- `已完成`
- `已搁置`

### 4.3 需求记录字段

每条需求建议记录以下信息：

- 编号
- 标题
- 类型
- 描述
- 优先级
- 状态
- 验收标准
- 备注/依赖
- 创建时间
- 最近更新时间
- 当前负责人（人类 / AI / 共同）

### 4.4 需求维护约定

- 人类可以直接补充/调整需求
- AI 负责帮助拆分、整理、补全验收标准、关联依赖
- 每个需求尽量只表达一个明确目标
- 如果一个需求过大，应拆分为多个可验收子需求
- 开发完成后应及时更新状态

---

## 5. 业务功能需求清单

### REQ-F-001 登录功能完善

- 类型：功能需求
- 状态：已完成 / 持续优化
- 描述：支持用户名、密码登录，完成基础认证闭环。
- 验收标准：
  - 输入正确用户名密码后可登录
  - 错误信息可见
  - 未登录用户无法访问受保护页面

### REQ-F-002 注册功能

- 类型：功能需求
- 状态：待开发
- 描述：支持新用户注册账号。
- 备注：当前模板已预留 `register.html`，但尚未实现。
- 验收标准：
  - 输入合法用户名密码后可创建用户
  - 重复用户名会被拒绝
  - 注册后可直接登录

### REQ-F-003 问数入口

- 类型：功能需求
- 状态：待开发
- 描述：用户可通过自然语言向系统提问，系统根据授权数据源返回答案。
- 验收标准：
  - 存在问数页面
  - 可提交问题
  - 系统能够返回文本答案或结构化结果

### REQ-F-004 后台用户管理

- 类型：功能需求
- 状态：待开发
- 描述：后台可查看、创建、编辑、禁用用户。
- 验收标准：
  - 支持用户列表
  - 支持用户状态管理
  - 支持基础筛选和搜索

### REQ-F-005 权限管理

- 类型：功能需求
- 状态：待开发
- 描述：不同角色访问不同功能。
- 验收标准：

### REQ-F-006 功能管理

## 7. 本地前端组件接入需求

### 7.1 目标组件

以下组件将作为后续前端开发基础设施：

- Bootstrap 5.3.8 Dist
- Font Awesome Free 5.15.4 Web
- layui 2.13.6

### 7.2 使用原则

- 组件必须本地化
- 解压后以静态资源方式使用
- 统一由项目本地目录管理
- 后续页面开发优先复用已有本地组件样式与能力
- 禁止在正式页面中直接引用互联网资源

### 7.3 组件接入优先级

建议优先级：

1. Bootstrap：统一基础布局和响应式栅格
2. Font Awesome：统一图标资源
3. layui：用于表单、表格、弹窗、管理后台交互

### 7.4 当前状态说明

- 三个组件已完成本地化解压
- 后续页面开发可直接在本地静态目录中引用
- 引用方式应保持统一，避免页面中散落多套资源路径
